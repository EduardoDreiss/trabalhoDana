Sobre o Projeto: Demonstração de Algoritmos de Busca

Este projeto foi desenvolvido com o objetivo de demonstrar, de forma prática e visual, o impacto da escolha da estrutura de dados e do algoritmo de busca na eficiência e performance de uma aplicação em grande escala.

O Dataset Spotify e o Desafio

Para simular um ambiente de aplicação real, foi utilizado um dataset com aproximadamente 28.000 músicas do Spotify. Este volume de dados é crucial para ilustrar como a complexidade temporal (O(N), O(log N), O(1)) se comporta diante de um catálogo extenso, refletindo os desafios enfrentados por plataformas de streaming.

Análise dos Algoritmos de Busca Implementados

  • Busca Sequencial (Linear):
    Metodologia O(N). A busca é feita item por item. Ideal para arrays não ordenados e pequenos volumes, mas sua eficiência cai drasticamente em um dataset de 28 mil itens.
  • Busca Indexada (Binária):
    Metodologia O(log N). Requer que o array esteja previamente ordenado (indexado) pelo track_id. Demonstra um salto de performance significativo ao dividir o espaço de busca pela metade em cada iteração.
  • Busca por Hash Map (Dicionário):
    Metodologia O(1) (Tempo Constante). Representa a forma mais rápida de busca, pois o acesso ao dado é feito diretamente através de um mapeamento de chave (ID) para valor. É a solução mais utilizada em sistemas que exigem recuperação instantânea de dados.

O objetivo final é permitir que o usuário compare diretamente o tempo de execução e o número de passos de cada algoritmo, validando a teoria da Complexidade de Algoritmos na prática.
